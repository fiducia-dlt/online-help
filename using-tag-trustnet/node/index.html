Node

The Node screen is only available in vendor nodes.

The primary objectives of this screen are to provide a vendor (ad server, CV tool, DSP, exchange, or publisher) a status of data being uploaded to their node, and insights on the data feeds that are coming from this node. It can be used to troubleshoot problems if data does not appear to be available in advertiser or agency nodes.

Member Node information

The top part of this screen provides information about the vendor log level data being uploaded to the vendor node. This information can be used to verify that this process is working correctly. The following provides a description of the information in each field.

Field Name Description
Last data upload The date and time that data was last uploaded to this vendor node.
Yesterday The number of records that were uploaded to this node yesterday.
Last 7 days The number of records that were uploaded to this node in the previous 7 days.

Data Sharing Activity

Each row in this table represents an individual feed of data going from the vendor node to another TAG TrustNet node (typically an advertiser or agency node). The following describes the information that appears in each column.

Column Description
Name The name of the advertiser or agency that the data feed is going to.
Member ID The TAG TrustNet member ID of that advertiser or agency of the node that the data feed is going to.
Native ID The ID used by the vendor for the adveriser or agency that the data feed is going to.
TAG-ID The TAG-ID of that advertiser or agency node that the data feed is going to.
Type The type of node that the data feed is going to (advertiser or agency)
Records shared last 7 days The number of records that were sent by the vendor node to the advertiser or agency node in the previous seven days.
